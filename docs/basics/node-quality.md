# 节点质量判断

拿到一批节点，怎么判断哪个好用、哪个是垃圾？

---

## 节点质量的核心维度

### 1. 延迟（Latency）

延迟是数据从你的设备到代理服务器来回一次的时间，单位毫秒（ms）。

| 延迟范围 | 体验 |
|---------|------|
| < 80ms | 优秀，几乎无感知 |
| 80–150ms | 良好，日常使用流畅 |
| 150–300ms | 一般，网页可用，视频可能卡顿 |
| > 300ms | 较差，明显延迟感 |
| 超时/无法连接 | 节点不可用 |

!!! tip "怎么测延迟？"
    在 Clash Verge / ClashMi 中，进入节点列表，点击闪电图标即可测速。测试的是你到代理服务器的延迟，不是到最终目标网站的延迟。

---

### 2. 速度（带宽）

延迟低不等于速度快。延迟衡量的是"响应速度"，带宽衡量的是"传输容量"。

一个延迟 30ms 但带宽只有 1Mbps 的节点，看 1080P 视频会卡；
一个延迟 150ms 但带宽 100Mbps 的节点，视频反而流畅。

**测速方法：**

- 连上节点后，访问 [fast.com](https://fast.com) 或 [speedtest.net](https://www.speedtest.net) 测试实际带宽
- 或在客户端中使用内置测速功能（部分客户端支持）

---

### 3. 稳定性

一个好节点应该：
- 长时间连接不断线
- 延迟波动小（抖动低）
- 高峰期（晚上 8-11 点）不明显变慢

**如何判断稳定性：**
- 在高峰期测试（不要只在凌晨测）
- 连接后持续使用 10-15 分钟，观察是否频繁断线
- 多次测延迟，看数值是否稳定（波动 > 100ms 说明不稳定）

---

### 4. 节点纯净度

"纯净度"指的是这个节点的 IP 有没有被各大网站和服务列入黑名单或限制名单。

**常见的"不纯净"表现：**

- Netflix 显示"该服务在你所在地区不可用"
- OpenAI/ChatGPT 提示无法在你的地区使用
- Google 频繁出现验证码（reCAPTCHA）
- 访问某些网站被直接跳转或拒绝
- 注册账号时被拒绝或需要额外验证

**为什么会不纯净？**

节点 IP 被大量用户同时使用，或之前被用于垃圾邮件、爬虫等行为，导致 IP 声誉差，被各大数据库标记为"高风险 IP"或"代理 IP"。

---

## 纯净度检测工具

### 基础检测

| 工具 | 地址 | 检测内容 |
|------|------|---------|
| IP.SB | [ip.sb](https://ip.sb) | 当前 IP 及基本信息 |
| ipinfo.io | [ipinfo.io](https://ipinfo.io) | IP 归属、运营商、是否为代理 |
| Scamalytics | [scamalytics.com](https://scamalytics.com) | IP 欺诈风险评分 |
| IPQualityScore | [ipqualityscore.com](https://www.ipqualityscore.com/free-ip-lookup-proxy-vpn-test) | 综合风险评分 |

### 流媒体解锁检测

连上节点后，用以下工具检测该节点能解锁哪些流媒体：

- **[流媒体检测脚本](https://github.com/lmc999/RegionRestrictionCheck)**：检测 Netflix、Disney+、HBO Max 等
- **[Streaming Unlock Test](https://streamingtest.cc)**：在线检测

### 综合评分参考

```
Scamalytics 风险评分：
0–25 分    → 低风险，纯净度好
26–75 分   → 中等风险，部分服务可能限制
76–100 分  → 高风险，基本无法使用需要 IP 声誉的服务
```

## 快速判断一个节点好不好用

```
第一步：连接节点
    ↓
第二步：访问 ip.sb，确认 IP 已切换
    ↓
第三步：测延迟（客户端内测速）
    ↓
第四步：测实际带宽（fast.com）
    ↓
第五步：访问 scamalytics.com，查看欺诈风险评分
    ↓
第六步（可选）：运行流媒体检测脚本，查看解锁情况
```

---

## 为什么同一服务商的节点质量差异很大？

- **服务器位置不同**：物理距离近的节点延迟低
- **服务器带宽不同**：高配服务器带宽大
- **用户数量不同**：共享用户多的节点高峰期慢
- **IP 类型不同**：机房 IP 和住宅 IP 的解锁能力差异大（详见下一章）
- **维护状态不同**：部分节点可能长期未更新、IP 已被封但服务商未发现

!!! tip "建议收藏几个好节点"
    找到延迟低、纯净度高的节点后，在客户端中标记或固定，不要每次都随机选。
